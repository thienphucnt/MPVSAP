### Generating Subtitles Programmatically
To create subtitles, you need to iterate over the stream provided by `edge_tts.Communicate`, feed the relevant metadata (such as `SentenceBoundary`) into the `SubMaker` instance, and then export the result[3][4].

```python
import edge_tts
import asyncio

async def generate_subtitles(text, output_audio, output_subs):
    communicate = edge_tts.Communicate(text, "en-US-JennyNeural")
    submaker = edge_tts.SubMaker()
    
    with open(output_audio, "wb") as audio_file:
        async for chunk in communicate.stream():
            if chunk["type"] == "audio":
                audio_file.write(chunk["data"])
            elif chunk["type"] == "metadata":
                submaker.feed(chunk)
    
    # Save the subtitles (default is SRT format)
    with open(output_subs, "w", encoding="utf-8") as file:
        file.write(submaker.get_srt())
```

```python
import requests

def upload_to_transfersh(file_path):
    url = f"https://transfer.sh/{file_path.split('/')[-1]}"
    with open(file_path, 'rb') as f:
```

```python
import requests

def upload_to_gofile(file_path):
    # 1. Get the best available server
    server_resp = requests.get("https://api.gofile.io/getServer")
    server = server_resp.json()["data"]["server"]
    
    # 2. Upload the file
    url = f"https://{server}.gofile.io/uploadFile"
    with open(file_path, 'rb') as f:
        files = {"file": f}
        response = requests.post(url, files=files)
    
    # 3. Parse the response for the download page link
    data = response.json()["data"]
    return f"https://gofile.io/d/{data['code']}"

# Usage

```

### Common Causes & Solutions

#### 1. Incorrect PKCE Implementation
TikTok requires PKCE for many OAuth flows to enhance security[4][5]. If you are receiving this error, your `code_challenge` or `code_verifier` likely does not meet TikTok's specific requirements[6][4].

*   **The Algorithm:** TikTok requires `S256` as the `code_challenge_method`[7][4].
*   **The Hashing:** You must generate a `code_verifier` (a high-entropy random string, 43–128 characters), hash it using **SHA256**, and then convert the result into a **hexadecimal string**[6][4].
    *   *Note:* Some developers mistakenly use Base64URL encoding (common for other OAuth providers) instead of the **hexadecimal encoding** required by TikTok[6][8].
    *   **Correct Logic:** `code_challenge = SHA256(code_verifier).toString(Hex)`[4].

#### 2. Mismatched Parameters
Ensure that the `code_challenge` sent in your initial authorization request matches the `code_verifier` sent later when exchanging the authorization code for an access token[9][10].
*   **Authorization URL:** Must include `code_challenge` and `code_challenge_method=S256`[7][4].
*   **Token Exchange:** Must include the original `code_verifier`[9][10].

#### 3. Library/Framework Misconfiguration
If you are using an authentication library (like `next-auth`, `BetterAuth`, or others), the library might be configured to expect a standard OAuth flow that does not automatically handle TikTok's specific PKCE requirements[1][3].
*   **Check Provider Settings:** Ensure your provider configuration explicitly supports TikTok's PKCE flow[1]. If the library handles PKCE automatically, verify that you aren't accidentally overriding or omitting parameters in your `signIn` call[1].
